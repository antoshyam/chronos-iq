11. Preventive Actions
LDAP High Availability: Implement LDAP failover/redundancy for qed-ldap.qualcomm.com:636. A single LDAP server failure should not block all CRMBuilds jobs across the entire Artifactory fleet.
Artifactory LDAP Monitoring: Create a Datadog monitor on the "Could not sync ldap groups" log pattern for Artifactory. Alert immediately when LDAP sync failures begin — this issue went undetected for the entire build window.
Artifactory Group Caching: Configure Artifactory to cache LDAP group memberships with a reasonable TTL (e.g., 15–30 minutes) so that short LDAP outages do not immediately impact build authorization.
Install Datadog Agent on crm-ubuntu327: This host has zero observability coverage. Enroll it in Datadog alongside the other CRM Ubuntu agents (crm-ubuntu265, crm-ubuntu238) for consistent fleet coverage.
CloudBees Log Forwarding: Forward CloudBees step logs to Datadog or Splunk to enable centralized RCA without requiring direct host access.
Exit Code 0 Failure Alerting: Investigate why CloudBees classifies a step with exit code 0 as "failed" — ensure the VerifySrc script uses non-zero exit codes for failure conditions to improve diagnostic clarity.
LDAP Health Check in Pre-Build: Add a pre-build step that validates Artifactory connectivity and authorization before invoking VerifySrc, providing a clearer failure message when LDAP is unavailable.
