11. Preventive Actions
Automated Kerberos ticket renewal (highest priority)
Deploy k5start, msktutil --auto-update, or a systemd timer on all build farm hosts to automatically renew Kerberos TGTs before expiry. Set renewal at 75% of ticket lifetime. This single action would have prevented all three job failures.
Kerberos ticket expiry monitor — leading indicator
Create a Datadog log monitor: alert on first occurrence of [Errno 127] Key has expired on any QCDFS mount path. This fires before the CIFS cascade and gives operators time to renew credentials before jobs are impacted.
CIFS SessSetup error rate monitor — lagging indicator
Create a Datadog log monitor: alert when CIFS SessSetup = -13 errors exceed 100 occurrences within 5 minutes across any build host. Route to arriverci.admins PagerDuty.
Pre-flight QCDFS + Kerberos check in VerifySrc
Add a CloudBees pre-step that: (1) runs klist to verify Kerberos ticket validity, (2) tests QCDFS mount accessibility. Fail fast with a clear error: "QCDFS mount unavailable — Kerberos ticket expired. Contact arriverci.admins." This prevents the misleading "Compile" failure classification.
Fix VerifySrc exit code behavior
VerifySrc exits with code 0 on infrastructure failure, causing the build system to misclassify the failure as "Compile." The step should exit with a non-zero code (e.g., exit 1) when source files are inaccessible, enabling automated detection and correct classification.
Auto-incident creation for farm-wide outages
The QCDFS Kerberos outage (50,676+ errors, 24 hosts, 90+ minutes, 3+ confirmed job failures) was not declared as a formal incident. Implement auto-incident creation triggered by the CIFS error monitor, with automatic assignment to arriverci.admins.
Datadog agent deployment across all CRMBuilds agents
Mandate Datadog agent installation on all CloudBees build agents (crm-ubuntu*, crm-ubuntulv* pools) as a prerequisite for agent registration.
