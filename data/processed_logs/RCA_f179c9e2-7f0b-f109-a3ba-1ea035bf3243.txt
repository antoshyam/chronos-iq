7. Probable Root Cause
Primary: IO Contention During Compilation — Network-IO Storm Caused Compiler Failure

The Make step initiated compilation of the Snapdragon Auto HQX firmware while simultaneously performing large network transfers (~91MB inbound + ~414MB outbound). The concurrent network buffer flushes to disk competed with the compiler's file reads/writes, causing IO await to spike to 10.84ms and IO utilization to reach 17.2%. With CPU iowait at 3.63%, compiler processes were blocked waiting for IO — unable to read source files or write object files at the required rate. This IO starvation caused one or more compilation units to fail (missing input file, read error, or compiler timeout), ultimately triggering GNU Make exit code 4 (fatal build error). The build ran for 16m 54s with two compile bursts before the accumulated errors caused Make to abort.
